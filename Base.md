# Network Fundamentals

Сети — это просто объединение объектов, которые могут взаимодействовать друг с другом. В вычислительной технике сетевое взаимодействие распространяется на технологические устройства. Сеть может состоять как из двух устройств, так и из миллиарда. К устройствам относятся ноутбуки, камеры видеонаблюдения, светофоры, сельскохозяйственная техника и многое другое. Сети интегрированы в нашу повседневную жизнь.

## Интернет

Интернет — это гигантская сеть, состоящая из множества меньших сетей. Первая версия интернета появилась в рамках проекта **ARPANET** в 1960-х годах.

## IP-адрес

IP-адрес используется для идентификации хоста в сети в течение определенного времени. После этого адрес может быть переназначен другому устройству.

* IP-адрес состоит из четырёх октетов (например, `192.168.1.1`).
* IP-адреса **не могут использоваться одновременно более чем одним устройством в одной сети**.
* Существует два типа IP-адресов:

  * **Публичный** — для идентификации устройства в интернете.
  * **Частный** — для идентификации устройства внутри локальной сети.

### Версии IP

* **IPv4** использует 32-битные адреса, что даёт около 4,29 миллиарда адресов.
* **IPv6** использует 128-битные адреса, обеспечивая огромное количество адресов (2^128) и улучшенную эффективность работы сети.

## MAC-адрес

Каждое сетевое устройство имеет **физический сетевой интерфейс**, которому присваивается уникальный **MAC-адрес (Media Access Control)**.

* Формат: 12 шестнадцатеричных символов, разделённых двоеточиями (например, `a4:c3:f0:85:ac:2d`).
* Первые 6 символов обозначают производителя, последние 6 — уникальный идентификатор устройства.

## Ping

**Ping** — базовый инструмент для проверки сетевого соединения. Он использует пакеты **ICMP (Internet Control Message Protocol)** для измерения доступности и производительности соединения между устройствами.

## Коммутаторы

**Коммутатор (Switch)** — устройство, которое объединяет несколько сетевых устройств, подключаемых к портам коммутатора.

* Эффективнее концентратора, так как передает пакеты только нужному получателю.
* Порты: 4, 8, 16, 24, 32, 64 и более.
* Может соединяться с другими коммутаторами и маршрутизаторами для повышения надежности сети.

## Маршрутизаторы

**Маршрутизатор (Router)** соединяет разные сети и передает данные между ними.

* Основной механизм: **маршрутизация** — выбор оптимального пути для передачи данных между сетями.

## Подсетирование

Подсетирование — разделение сети на меньшие подсети, чтобы:

* Повысить эффективность,
* Обеспечить безопасность,
* Упростить управление сетью.

Пример: в компании могут быть подсети для:

* Бухгалтерии
* Финансового отдела
* HR

### Маска подсети

* Маска подсети — 32-битное число (например, `255.255.255.0`), определяющее, какие адреса входят в подсеть.
* Подсети используют IP-адреса для определения:

  1. **Адрес сети** — идентификатор подсети (например, `192.168.1.0`).
  2. **Адрес хоста** — идентификатор конкретного устройства (например, `192.168.1.100`).
  3. **Шлюз по умолчанию** — устройство для выхода в другие сети (например, `192.168.1.254`).

Пример: дома у вас одна подсеть, а в офисе — множество подсетей для разных отделов.

## ARP (Address Resolution Protocol)

ARP сопоставляет IP-адреса с MAC-адресами:

1. **ARP-запрос** — устройство спрашивает: «Кто имеет этот IP-адрес?»
2. **ARP-ответ** — устройство с нужным IP отвечает своим MAC-адресом.

* Результат сохраняется в **ARP-кэше** для ускорения последующих запросов.

## DHCP (Dynamic Host Configuration Protocol)

DHCP автоматически назначает IP-адрес устройствам в сети:

1. Устройство отправляет запрос на IP (DHCP Discover).
2. Сервер предлагает IP (DHCP Offer).
3. Устройство подтверждает выбор (DHCP Request).
4. Сервер завершает процесс (DHCP Acknowledge).

---

## 1. IP-адрес и подсеть

```
Подсеть: 192.168.1.0/24
┌───────────────────────────┐
│ Сеть 192.168.1.0          │
│ Маска: 255.255.255.0      │
├─────────────┬─────────────┤
│ Хост 1      │ 192.168.1.1 │
│ Хост 2      │ 192.168.1.2 │
│ ...         │ ...         │
│ Хост 254    │ 192.168.1.254│
└─────────────┴─────────────┘
Шлюз по умолчанию: 192.168.1.254
```

**Пояснение:**

* Адрес сети — идентификатор подсети.
* Адрес хоста — уникальный для устройства.
* Шлюз — устройство для выхода в другую сеть.

---

## 2. Работа ARP (Address Resolution Protocol)

```
Устройство A хочет отправить данные Устройству B

IP A: 192.168.1.10
MAC A: AA:AA:AA:AA:AA:AA

IP B: 192.168.1.20
MAC B: ??

[Устройство A] ---ARP Request---> [Сеть]
"Кто имеет IP 192.168.1.20? Отправьте свой MAC!"

[Устройство B] ---ARP Reply---> [Устройство A]
"Я 192.168.1.20, мой MAC = BB:BB:BB:BB:BB:BB"

Устройство A теперь знает MAC B и может отправлять данные напрямую.
```

---

## 3. DHCP-процесс (автоматическое получение IP)

```
Устройство C подключается к сети:

1. DHCP Discover (C --> Broadcast)
   "Есть ли DHCP-сервер?"

2. DHCP Offer (Server --> C)
   "Вот IP: 192.168.1.100, маска: 255.255.255.0, шлюз: 192.168.1.254"

3. DHCP Request (C --> Server)
   "Я хочу этот IP"

4. DHCP Acknowledge (Server --> C)
   "Хорошо, IP подтвержден. Можете использовать"

Теперь устройство C готово к работе в сети.
```

---

## 4. IP и маршрутизация (схематично)

```
[Устройство A] 192.168.1.10
        │
        │ (192.168.1.10 -> 192.168.2.20)
        ▼
   [Маршрутизатор 1]
        │
        │ (маршрутизация пакета)
        ▼
[Сеть 192.168.2.0/24]
        │
        ▼
[Устройство B] 192.168.2.20
```

**Пояснение:**

* IP-адреса используются для выбора маршрута.
* Маршрутизатор знает, куда передавать пакеты между подсетями.

# OSI Model

Модель **OSI (Open Systems Interconnection)** — это концептуальная модель, описывающая, как данные передаются в сети. Она помогает понять процессы маршрутизации, инкапсуляции и взаимодействия устройств с разными функциями и конструкциями.

Главное преимущество модели OSI в том, что устройства, соответствующие её стандартам, могут понимать и интерпретировать данные друг друга независимо от производителя или технологии.

Модель OSI состоит из **семи уровней**, начиная с уровня 1 (физического) и до уровня 7 (прикладного). На каждом уровне данные обрабатываются, и к ним добавляются специальные заголовки или метаданные — процесс называется **инкапсуляцией**.

---

## Уровень 1: Физический (Physical Layer)

* Отвечает за физические компоненты сети и передачу электрических или оптических сигналов.
* Примеры: кабели, разъёмы, повторители, хабы.
* Задача: передавать "сырые" биты от одного устройства к другому.

---

## Уровень 2: Канальный (Data Link Layer)

* Основная задача — **физическая адресация** и обеспечение надежной передачи данных.
* Добавляет **MAC-адрес** отправителя и получателя к данным.
* Включает контроль ошибок и управление потоком.
* Пример устройств: коммутаторы (switches).

---

## Уровень 3: Сетевой (Network Layer)

* Отвечает за **маршрутизацию данных** между сетями.
* Работает с IP-адресами, определяет оптимальный путь для пакетов.
* Примеры протоколов: **IP**, **OSPF**, **RIP**.
* Примеры устройств: маршрутизаторы (routers).

Факторы выбора маршрута:

1. Кратчайший путь (меньше промежуточных устройств).
2. Надежность маршрута (минимальные потери пакетов).
3. Скорость физического соединения (медный кабель vs. оптоволокно).

---

## Уровень 4: Транспортный (Transport Layer)

* Обеспечивает **доставку данных между приложениями**.
* Протоколы:

  * **TCP (Transmission Control Protocol)** — надежная доставка с подтверждением.
  * **UDP (User Datagram Protocol)** — быстрый, но ненадежный.
* Управляет сегментацией данных и проверкой целостности.

---

## Уровень 5: Сессионный (Session Layer)

* Устанавливает, поддерживает и завершает соединения между приложениями.
* Контролирует длительность сеансов и их уникальность.
* Позволяет использовать **контрольные точки**, чтобы при потере данных пересылать только недостающие сегменты.

---

## Уровень 6: Представительский (Presentation Layer)

* Преобразует данные в формат, понятный получающему устройству.
* Реализует стандартизацию данных независимо от используемого программного обеспечения.
* Функции безопасности: **шифрование и дешифрование** данных (например, HTTPS).
* Примеры: преобразование форматов текста, изображений, видео.

---

## Уровень 7: Прикладной (Application Layer)

* Уровень, с которым взаимодействует пользователь.
* Предоставляет интерфейс и протоколы для работы приложений:

  * Веб-браузеры (HTTP/HTTPS)
  * Почтовые клиенты (SMTP, IMAP, POP3)
  * FTP-клиенты (FileZilla)
  * DNS (преобразование доменных имен в IP-адреса)

---

## Важные концепции

* **Инкапсуляция**: добавление заголовков и информации на каждом уровне.
* **Декapsulation**: удаление заголовков на приёмной стороне для извлечения данных.
* Каждый уровень OSI выполняет свои функции независимо, что упрощает проектирование сетей и взаимодействие различных устройств.

---

```
+------------------------------------------------+
| Level 7: Application Layer                     |
| - Пользовательские приложения                 |
| - HTTP, FTP, SMTP, DNS                         |
+------------------------------------------------+
| Level 6: Presentation Layer                    |
| - Форматирование и преобразование данных      |
| - Шифрование/дешифрование (например, HTTPS)   |
+------------------------------------------------+
| Level 5: Session Layer                         |
| - Управление сессиями                          |
| - Установка, поддержка и завершение соединений|
+------------------------------------------------+
| Level 4: Transport Layer                        |
| - Надежная доставка данных (TCP)              |
| - Быстрая доставка без подтверждения (UDP)    |
| - Сегментация и проверка целостности          |
+------------------------------------------------+
| Level 3: Network Layer                          |
| - Маршрутизация данных                         |
| - Работа с IP-адресами                         |
| - OSPF, RIP                                    |
+------------------------------------------------+
| Level 2: Data Link Layer                        |
| - Физическая адресация (MAC)                   |
| - Контроль ошибок и управление потоком         |
| - Коммутаторы (Switches)                       |
+------------------------------------------------+
| Level 1: Physical Layer                         |
| - Передача "сырых" битов                        |
| - Кабели, разъемы, повторители, хабы          |
+------------------------------------------------+
```

# 🔥 Визуальное правило (самое важное)

```plaintext id="core_rule"
IP  → НЕ меняется (кроме TTL)
MAC → меняется на каждом hop
```

❯ Заголовок между L2 и L3 (L2.5 / Shim Header)
В некоторых сетевых технологиях между заголовками канального (L2) и сетевого (L3) уровней может вставляться дополнительный заголовок.
Такой заголовок не относится строго ни к одному уровню модели OSI и часто называется:
L2.5
Shim header
📌 Shim header — это «промежуточный» заголовок, который вставляется между L2 и L3 для добавления дополнительной информации при передаче пакетов.

❯ MPLS как пример Shim Header
Наиболее известный пример — MPLS (Multiprotocol Label Switching).
MPLS используется для:
ускорения маршрутизации (label switching вместо IP lookup)
управления трафиком (Traffic Engineering)
построения VPN (L2VPN, L3VPN)
📌 MPLS вставляется:
L2 → MPLS → L3

❯ Как выглядит Ethernet + MPLS
+-------------------+-------------------+-----------+-------------------+-------------------+--------+
| Destination MAC   | Source MAC        | EtherType | MPLS Label Stack  | Payload (L3)      | FCS    |
| 6 bytes           | 6 bytes           | 2 bytes   | N * 4 bytes       | (IP packet)       | 4 bytes|
+-------------------+-------------------+-----------+-------------------+-------------------+--------+
📌 EtherType для MPLS:
0x8847 — unicast
0x8848 — multicast

❯ Стек меток MPLS (Label Stack)
👉 Важно: MPLS может содержать несколько меток, а не одну.
Label 1
Label 2
Label 3
...
Каждая метка = 4 байта

❯ Структура MPLS заголовка
+----------------------+--------------+-----+--------+
| Label (20 bits)      | TC (3 bits)  | S   | TTL    |
+----------------------+--------------+-----+--------+

❯ Поля MPLS
Label (20 бит)
используется для пересылки пакета
маршрутизаторы смотрят только на label (а не на IP)

Traffic Class (TC) (3 бита)
(раньше назывался EXP)
используется для QoS
аналог приоритизации (как PCP в VLAN)

S (Bottom of Stack) (1 бит)
1 → последняя метка
0 → есть ещё метки
📌 Это важно для:
MPLS VPN
MPLS tunneling

TTL (8 бит)
аналог TTL в IP
предотвращает зацикливание

❯ Пример MPLS стека
[MPLS Label 100 | S=0]
[MPLS Label 200 | S=1]
[IPv4 Packet]
📌 Это означает:
верхняя метка — транспортная
нижняя — сервисная (например VPN)

❯ Где используется MPLS
провайдерские сети (ISP)
MPLS VPN (L3VPN, L2VPN)
Traffic Engineering
QoS
📌 В обычных LAN/MPLS не используется — это технология уровня провайдеров.

❯ Важное уточнение
MPLS:
не заменяет IP
работает поверх L2 и перед L3
может инкапсулировать:
IPv4
IPv6
Ethernet (L2VPN)

Shim header — это промежуточный заголовок между L2 и L3
