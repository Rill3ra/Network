#  Layer 2 (Data Link Layer) — Полный и детализированный конспект

##  Основные функции L2 (Data Link Layer)
Уровень 2 модели OSI отвечает за доставку данных внутри одного **локального сегмента сети**. L2 работает с **кадрами** (frames), использует **MAC-адреса** и управляет доступом к среде передачи.

### Основные задачи L2:
1. **Доставка данных** — между устройствами внутри одного сегмента.
2. **Использование MAC-адресов** — для идентификации устройств на уровне L2.
3. **Ошибка контроль (FCS/CRC)** — обнаружение ошибок при передаче данных.
4. **Изоляция и сегментация сети** — через VLAN.
5. **Обнаружение соседей** — через ARP (Address Resolution Protocol).
6. **Управление доступом к каналу** — с использованием различных механизмов, таких как CSMA/CD.

---

##  Ethernet Frame (структура Ethernet кадра)

Основной формат передачи данных в L2 — это **Ethernet кадр**. Кадры содержат следующие части:

1. **Destination MAC (6 байт)** — MAC-адрес получателя.
2. **Source MAC (6 байт)** — MAC-адрес отправителя.
3. **EtherType (2 байта)** — указывает, какой протокол используется в данных (например, IPv4 или ARP).
4. **Payload (переменная длина)** — данные уровня L3 (например, IP-пакет).
5. **FCS (Frame Check Sequence)** — контрольная сумма для проверки целостности данных.

### Пример структуры Ethernet кадра:
```

+----------------+----------------+------------+------------+----------------+
| Destination MAC| Source MAC     | EtherType  | Payload    | FCS            |
+----------------+----------------+------------+------------+----------------+

```

---

##  MAC-адрес (Media Access Control)

### Что это?
MAC-адрес — это уникальный идентификатор для каждого сетевого интерфейса устройства. Этот адрес используется для адресации в пределах **локальной сети**.

### Формат MAC-адреса:
```

AA:BB:CC:DD:EE:FF

```
Это 48 бит (6 байт), из которых:
- Первые 3 байта — **OUI (Organizationally Unique Identifier)**, выделяются производителю.
- Последние 3 байта — уникальный идентификатор интерфейса устройства.

---

##  Коммутатор (Switch) на L2

### Основная функция:
Коммутатор работает на L2, и его основная задача — **передача кадров по MAC-адресам**.

1. Коммутатор **изучает MAC-адреса** в процессе передачи кадров.
2. Он **записывает** в свою таблицу, какой MAC-адрес связан с каким портом.
3. Когда он получает кадр с MAC-адресом назначения, он ищет его в своей таблице и пересылает кадр через соответствующий порт.

### CAM / MAC Table:
Это таблица, в которой хранится соответствие:
```

MAC-адрес → Порт

```

---

##  Типы трафика L2

### 1. **Unicast** — Один → Один
- Стандартный тип передачи: от одного устройства к другому.

### 2. **Broadcast** — Один → Все
- Трафик, который отправляется всем устройствам в пределах broadcast domain.
- MAC-адрес: `FF:FF:FF:FF:FF:FF`
- Используется в таких протоколах, как ARP и DHCP.

### 3. **Multicast** — Один → Группа
- Трафик, предназначенный для нескольких устройств одновременно.
- Например, используется для видеоконференций или IPTV.

---

##  Broadcast Domain

**Broadcast Domain** — это группа устройств, которые могут получать **broadcast** сообщения от других устройств в пределах этой же сети. Коммутатор **не пересылает broadcast за пределы своего broadcast domain**. Каждый VLAN является отдельным broadcast domain.

### Пример:
Если устройства находятся в разных VLAN, то **broadcast** сообщения не могут быть переданы между ними без помощи маршрутизатора.

---

##  VLAN (Virtual LAN)

**VLAN** (виртуальная локальная сеть) — это логическое разделение физической сети на несколько отдельных виртуальных сетей.

### Основные свойства:
- Каждый **VLAN** представляет собой **отдельный broadcast domain**.
- VLAN позволяет изолировать трафик между разными группами устройств.
- VLAN работает только на **L2**, чтобы разделить сеть на части, но для связи между VLAN необходимо использовать маршрутизатор (L3).

### Пример:
- **VLAN 10**: `192.168.10.0/24`
- **VLAN 20**: `192.168.20.0/24`

### Зачем нужно разделение на VLAN?
- **Безопасность**: ограничение доступа между группами пользователей.
- **Сегментация трафика**: уменьшение количества широковещательных пакетов в сети.
- **Управление трафиком**: возможность управления полосой пропускания и приоритетами для разных типов трафика.

---

##  Access vs Trunk Ports

### 1. **Access Port**
- Принадлежит **одному VLAN**.
- Трафик на access порту **не имеет тегов**.
- Обычно используется для подключения конечных устройств, таких как ПК или принтеры.

### 2. **Trunk Port**
- Может передавать трафик для **нескольких VLAN**.
- Использует **тегирование** для различения VLAN (например, через стандарт 802.1Q).
- Используется для соединения **коммутаторов** или **коммутаторов с маршрутизаторами**.

---

## ARP (Address Resolution Protocol)

**ARP** — это протокол, который используется для преобразования **IP-адреса** в **MAC-адрес**.

### Как работает ARP:
1. **ARP Request** — устройство отправляет **broadcast запрос**, чтобы узнать, какой MAC-адрес соответствует определённому IP-адресу.
2. **ARP Reply** — устройство с нужным IP-адресом отвечает **unicast** сообщением с своим MAC-адресом.
3. MAC-адрес записывается в **ARP-таблицу** устройства.

### Ограничения ARP:
- ARP работает только в рамках одного **broadcast domain**.
- Он **не маршрутизируется**.

---

## 🔥 Flooding (Неизвестный Unicast)

Если коммутатор не знает, на какой порт отправить кадр с данным MAC-адресом, он **флудит** кадр на все порты в соответствующем VLAN. Это называется **flooding**.

---

##  STP (Spanning Tree Protocol)

**STP** (Spanning Tree Protocol) используется для предотвращения **L2 петель (loops)** в сети.

### Почему важен STP?
Без STP петли могут привести к:
- **Broadcast storm** (шторм широковещательных сообщений)
- **MAC flapping** (неправильная запись MAC-адресов)
- Перегрузка сети

### Как работает STP:
- STP строит **дерево топологии**, исключая лишние связи, чтобы предотвратить петли.
- Все порты в сети находятся в одном из 5 состояний:
  1. **Blocking** — порт заблокирован для предотвращения петель.
  2. **Listening** — порт слушает трафик, но не передаёт его.
  3. **Learning** — порт изучает MAC-адреса.
  4. **Forwarding** — порт передаёт данные.
  5. **Disabled** — порт отключён.

### Типы STP:
- **Classic STP** (802.1D)
- **Rapid Spanning Tree Protocol (RSTP)** (802.1w)

---

## 🔄 L2 vs L3 (Ключевая разница)

| Уровень | Описание | Пример |
|---------|----------|--------|
| **L2**  | Доставка данных внутри одного сегмента сети с использованием MAC-адресов. | Коммутатор пересылает кадры. |
| **L3**  | Маршрутизация между сетями с использованием IP-адресов. | Маршрутизатор пересылает пакеты. |

## ⚠️ Частые ошибки

### 1. **VLAN ≠ IP-сеть**
- VLAN — это L2 разделение сети, но оно не связано напрямую с IP-сетями.
- Например, несколько VLAN могут иметь одну IP-сеть, но это не всегда хорошо с точки зрения архитектуры.

### 2. **Trunk не даёт маршрутизацию**
- Trunk позволяет передавать несколько VLAN, но маршрутизация между ними требует L3 устройства (маршрутизатора или L3 switch).

### 3. **ARP ≠ определение сети**

* ARP работает на L2, и определяет MAC-адрес для IP в одном broadcast domain. Он не проверяет принадлежность устройства к подсети.

---

##  Резюме и Основные принципы L2

1. **MAC-адрес** — уникальный идентификатор устройства на L2.
2. **Коммутатор** — пересылает кадры на основе MAC-адресов и обучается в процессе передачи.
3. **VLAN** — логическое разделение сети для управления broadcast domain.
4. **ARP** — сопоставляет IP-адреса с MAC-адресами в одном broadcast domain.
5. **STP** — предотвращение петель в сети.
6. **Access и Trunk порты** — управление трафиком внутри VLAN и между VLAN.

---

#  Итог

Layer 2 (L2) отвечает за работу с кадрами и MAC-адресами в пределах локальной сети, включая управление трафиком через VLAN, использование ARP для разрешения адресов и защиту от петель с помощью STP. Этот уровень также обеспечивает изоляцию трафика внутри сети и сегментирует её на различные broadcast domains.
